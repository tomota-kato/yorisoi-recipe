## Phase 1: 技術検証 (Proof of Concept)

**目的:** プロジェクトの最も重要なコア機能であり、技術的な不確実性が高い「YouTubeリンクからのAIレシピ生成」が実現可能かを確認する。

-   **タスク:**
    -   [ ] **YouTube Data APIの検証:**
        -   特定の動画URLから、タイトル、説明、字幕（文字起こし）データが取得できるかを確認する。
        -   簡単なPythonスクリプトでAPIを叩いてみる。
    -   [ ] **Generative AI APIの検証:**
        -   YouTubeから取得したテキスト情報をインプットとして、Generative AI API (Google Gemini等を想定) に渡し、「レシピ名」「材料リスト」「手順」を構造化データ（JSON形式など）で抽出できるかを検証する。
        -   プロンプトエンジニアリングを試行し、期待する形式で出力させる。
    -   [ ] **検証結果の評価:**
        -   AIによる抽出精度は実用に足るレベルか、どの程度のユーザーによる修正が必要になりそうかを評価する。
        -   APIのコストやレスポンスタイムを見積もる。

---

## Phase 2: MVP - 基本的なレシピ管理機能 (CRUD)

**目的:** ユーザーが自身のレシピブックとして使える、最も基本的な基盤を構築する。

-   **タスク:**
    -   [ ] **プロジェクトセットアップ:**
        -   Djangoプロジェクトの初期設定。
        -   フロントエンド（HTML/CSS/JS）の基本的なファイル構成を作成。
    -   [ ] **ユーザー認証機能:**
        -   サインアップ、ログイン、ログアウト機能を実装する。
    -   [ ] **データベースモデル設計と実装:**
        -   `要件定義.md`に基づき、`Users`, `Recipes`, `Ingredients`, `Recipe_Ingredients`モデルをDjangoで作成。
        -   `manage.py makemigrations` および `migrate` を実行。
    -   [ ] **レシピのCRUD機能:**
        -   レシピの手動での作成(Create)、一覧・詳細表示(Read)、更新(Update)、削除(Delete)ができるAPIと画面を実装する。
        -   最低限のフォームとビューを作成する。

---

## Phase 3: MVP - コア機能の実装 (AIレシピ作成)

**目的:** Phase 1の技術検証の結果をアプリケーションに統合し、本アプリの最大の価値であるAIレシピ登録機能をユーザーに提供する。

-   **タスク:**
    -   [ ] **AIレシピ登録画面の実装:**
        -   YouTubeのURLを入力するフォームをフロントエンドに作成。
    -   [ ] **バックエンドロジックの実装:**
        -   フロントエンドからURLを受け取り、外部API（YouTube, Generative AI）と連携してレシピ情報を生成するDjangoのビューを実装する。
    -   [ ] **結果の表示と編集機能:**
        -   AIが生成したレシピ情報をフォームに自動入力し、ユーザーが確認・修正できるようにする。
        -   修正した内容をデータベースに保存する機能を実装する。

---

## Phase 4: MVP - 献立作成機能

**目的:** 登録したレシピを活用し、日々の計画を立てられるようにする。

-   **タスク:**
    -   [ ] **データベースモデルの追加:**
        -   `Weekly_Menus`, `Weekly_Menu_Recipes` モデルを実装・マイグレーション。
    -   [ ] **週単位の献立作成機能:**
        -   シンプルなカレンダーUI（まずは1週間分）を実装する。
        -   登録済みのレシピを特定の日付に割り当てる機能を実装する。（MVPではドラッグ＆ドロップではなく、日付とレシピを選択するシンプルな形式でも可）
    -   [ ] **献立の表示機能:**
        -   作成した週の献立をカレンダー上で確認できるようにする。

## Phase 5: MVP - 買い物リスト生成機能

**目的:** 献立から買い物を支援する機能を実装し、MVPを完成させる。

-   **タスク:**
